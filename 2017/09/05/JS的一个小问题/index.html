JS的一个小问题

发表于 2017-09-05 | |
  1. 开发中不可避免的会接触到 JavaScript。
  2. 像我们这样的小公司没有对应的人跟你配合

在这样的情况下：
不懂点 JavaScript 可能都不能好好写 Objective-C 。

问题描述

有个以前同事告诉我，他遇到一个奇怪的事情。在调用js代码的时候，页面好像假死了。我很好奇，然后我就顺着他给我描述的问题，写了一个demo，看看能不能复现。

代码主要就是他有一个 UINavigationController 控制器。在他的根 A 的页面上有一个 UIWebView 页面用来加载业务。点击导航栏的 rightBarButtonItem ，会把另一个业务页面 B push 进来。

问题就发生在对 B 页面做 pop 操作的时候，他在 A 的 viewXXXAppear 中添加了alert()函数。然后他的页面就卡住了。

alert 的问题

前一阵子，公司做 RN 浏览器项目时候，对 js 稍微有点了解，知道 js 引擎是基于事件驱动单线程执行的。并且在 iOS 上执行 js 代码，必须把它放到主线程中操作。因为如果不这样的话，他就会在次要线程中调用 UIKit，可能会导致 Crash。

那么这里viewXXXAppear中显然是主线程的调用，为什么会卡主？

因为 js 是单线程的，而在 js 中 alert 会阻断代码的执行。只有在响应完 alert ，代码才会继续执行。
